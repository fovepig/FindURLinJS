FindURLinJS-AI Pro 🚀
FindURLinJS-AI Pro 是一款基于 hanbufei/findUriInjs 进行深度二次开发的 Burp Suite 插件。
在原始版本通过正则表达式提取链接的基础上，本项目引入了 大语言模型 (LLM) 技术，实现了对 JS/HTML 源码的智能解析。它不仅能发现隐藏的 API 接口，还能通过 AI 理解代码上下文，自动给出接口的用途说明和风险标签，是渗透测试中信息收集阶段的“核武器”。
🌟 核心特性
智能 AI 引擎：对接自定义后端 AI 接口，利用 LLM 的语义理解能力，从复杂的 JS 逻辑中提取潜在的 URL 和 API 节点。
用途与风险识别：AI 会自动分析每个接口的功能（如：登录、上传、配置、获取用户信息）并打上标签（如：[关键页面]、[敏感接口]）。
主从表联动 UI：
嗅探列表 (Master)：实时列出捕获到的 JS/HTML 目标文件及其包含的链接总数。
嗅探详情 (Detail)：点击文件即可查看该文件内所有链接的路径、用途和 AI 判定的标签。
多维过滤规则库：
请求过滤：内置针对 Web、微信小程序、REST API 的高频正则，精准拦截目标流量。
响应匹配：内置针对 path、token、admin 等敏感关键字的预匹配逻辑，节省 AI 额度。
规则推荐系统：内置海量实战推荐规则，支持双击快速添加，支持自定义规则弹窗配置。
配置持久化：支持自定义 AI 后端基础 URL，并自动保存配置，重启 Burp 无需重填。
🛠️ 项目架构
本插件采用 客户端-服务端 架构：
客户端 (本插件)：作为 Burp Suite 插件运行，负责流量监听、源码采集、规则过滤和结果展示。
服务端 (需自行搭建)：基于 Spring Boot + Spring AI 搭建。接收源码，调用大模型（如 GPT-4, DeepSeek, Ollama 等）处理并返回 JSON 格式的识别结果。
📥 安装指南
1. 编译项目
克隆本项目后，使用 Maven 进行编译：
code
Bash
mvn clean package
编译完成后，在 target 目录下找到 finurlinsplus-2.0-jar-with-dependencies.jar。
2. 加载插件
打开 Burp Suite -> Extensions -> Installed。
点击 Add -> 选择 Java。
选择打包好的 jar-with-dependencies.jar 文件加载。
3. 环境要求
JDK: 17+
Burp Suite: 2023.x 及以上版本（推荐）
⚙️ 使用说明
配置中心
AI 基础 URL：填写你本地或服务器运行的 AI 后端接口地址（例如：http://127.0.0.1:8899/api/chat/）。
管理规则：
在下方的“推荐模式”中双击常用规则，即可快速加入生效列表。
点击“自定义添加规则”，在弹窗中输入你自己的正则逻辑。
点击“保存配置”持久化所有设置。
嗅探结果
正常浏览目标网站，符合规则的 JS/HTML 会自动发送给 AI。
在“嗅探结果”选项卡，上方查看文件概览。
点击任意一行，下方会展示 AI 深度解析出的链接详情。
📊 推荐规则示例
场景	请求过滤正则 (双击添加)	用途
Web 渗透	(^GET /.*\.js)|(^GET /.*\.html)	采集网页所有静态资源和页面
小程序测试	.*wechat.*\.js	专门针对微信侧业务 JS 进行提取
API 监控	(^POST /api/.*)|(^GET /api/.*)	监控并解析所有 RESTful 接口
敏感发现	token|auth|key|secret	响应匹配中自动高亮含鉴权信息的代码
